ESCA Kit Barcode System
Executive Overview  ·  Barcode-driven inventory and checkout for CTE career kits
What This System Does

Dallas ISD CTE ESCA career kits are loaned to campus counselors to support career exploration programming. This system tracks every kit — and every item inside it — from the moment a barcode label is printed through checkout, return, and audit. All data writes automatically to a Google Sheet. No manual entry is required after labels are applied.

Streamlined standard workflow
No second sticker is ever applied to the kit case. The TipWeb tag number is entered as the ESCA kit barcode when the kit is created in Admin. From that point on, one scan handles TipWeb cross-reference and ESCA checkout tracking simultaneously.
TipWeb + ESCA — How They Work Together
TipWeb owns district asset inventory
TipWeb tracks that the kit physically exists and belongs to Dallas ISD. ESCA does not create duplicate TipWeb records — the TipWeb tag number is the ESCA barcode, so both systems stay in sync automatically with zero extra work.
ESCA owns the checkout workflow
ESCA tracks who has the kit, from which campus, in which region, when it went out, when it came back, and the condition of every item inside. This data surfaces in the Admin dashboard for leadership.
One scan, two systems — no duplication
Because the TipWeb tag number is entered as the ESCA kit barcode at setup, a counselor's single scan at checkout simultaneously records the ESCA loan and provides a direct TipWeb cross-reference. No second sticker. No manual reconciliation. No confusion.
End-to-End Workflow
Admin Setup — ESCA Staff (one time per kit)
1
Create career template
Define what belongs inside a kit type — e.g. EMT kit: stethoscope, BP cuff, gloves, penlight. Set reorder thresholds per item.
2
Create physical kit
Enter kit name, link to career template, and type in the TipWeb asset tag number as the ESCA scan barcode — this is the one number that drives both systems.
3
Generate item barcodes
Admin generates unique barcodes for every item inside the kit. Each barcode is permanently linked to the kit's TipWeb tag.
4
Print and apply item labels
Print the item barcode sheet and apply stickers to each item inside the case. No second sticker on the outside — the TipWeb tag is already there. Kit is ready for checkout.
Checkout — Counselor (at pickup)
5
Scan kit barcode
Counselor opens the Counselor Hub on any device and scans the ESCA barcode on the kit case.
6
Verify item checklist
System displays every item that should be in the kit with its status. Counselor acknowledges the list.
7
Select campus and confirm
Counselor selects their campus (auto-tags the Director Region) and completes checkout. Loan record written to Sheet.
Check-In — Counselor (at return)
8
Scan kit barcode
Same barcode, same Hub. System recognizes the kit is checked out and opens the return flow automatically.
9
Clean return
Counselor confirms non-consumable items are present. Consumables (batteries, gloves, etc.) are noted but not required.
10
Problem report
If anything is missing or damaged, counselor flags the item, selects the issue type, and adds notes. Sheet is updated automatically.

Frequently Asked Questions
Do we need to change anything in TipWeb?
No. TipWeb continues to work exactly as it does today. The TipWeb asset tag number that is already on every kit case is entered as the ESCA kit barcode at setup. From that point forward, the same number identifies the kit in both systems. ESCA does not create, modify, or delete any TipWeb records — the two systems remain fully independent and always in sync.
What if items from different kits get mixed up during a reset?
Each item barcode permanently stores the kit ID it was created for — which is the kit's TipWeb tag number. Scanning a misplaced item in the Counselor Hub immediately tells the counselor which kit it belongs to and shows the TipWeb tag so it can be physically located. During Admin audits, unexpected items are flagged automatically.
Can counselors use this on their phones?
Yes. The Counselor Hub is a web app that runs in any Chromium browser, including Chrome on a phone or tablet. A USB or Bluetooth barcode scanner also works — it inputs the barcode exactly like a keyboard.
Where does all the data go?
Every scan writes directly to a Google Sheet linked to the Admin Portal. Tabs include Kits, KitItems, Loans, CheckoutItems, CheckinIssues, Campuses, AuditLog, and Settings. No external database is required. ESCA staff can open the Sheet directly for custom reports at any time.
How does the regional dashboard work?
When a counselor checks out a kit, they select their campus from a dropdown. Each campus is assigned to one of the eight Dallas ISD Director Regions. The Admin Dashboard aggregates checkout data by region — showing total checkouts, open loans, and campuses served — with a bar chart for leadership review.
What email notifications are coming?
Two flows are planned: (1) an automatic confirmation to the counselor and campus principal on every checkout — personalized with kit, campus, career type, and return expectations; (2) a manual announcement blast ESCA staff can send to all counselors with open loans for return deadlines and pickup hours. Both are configured in Admin under Email Notifications once email verbiage is provided.
